Configuration

You will need to configure the LLM connection and authorizion secrets.

GenAIScript uses a .env file to store the secrets (this file should never be commited to your source control!).

  1. Create or update a .gitignore file in the root of your project and make it sure it includes .env. This ensures that you do not accidentally commit your secrets to your source control.

  2. Create a .env file in the root of your project.

    • .gitignore
    • .env

    Tip

    If the .gitignore file is properly configured, the .env file will appear grayed out in your source control.

  3. Update the .env file with the configuration information (see below).

To learn more about LLM configurations, see authorization information.

OpenAI

  1. Create a new secret key from the OpenAI API Keys portal.

  2. Update the .env file with the secret key.

Azure OpenAI

  1. Open your Azure OpenAI resource and navigate to the Keys and Endpoint tab.

  2. Update the .env file with the secret key and the endpoint.

Jan, LMStudio, OLLama

Jan, LMStudio, OLLama are desktop application that let you run model locally. They expose local API servers that compatible with GenAiScript.

Running tools locally may require additional GPU resources depending on the model you are using.

  1. Start Jan/LMStudio/OLLama, load your model and start the Local API Server.

  2. Update the .env file with the local server information.

VSCode Copilot

Visual Studio Code Insiders provides an experimental support to use your Copilot subscription to access a LLM.

Limitations

  • This feature is still a proposed api and requires the Insiders editor.
  • The API changes daily and may not be stable.
  • Functions and Images are not supported.
  • The model configuration, temperature, max tokens, may not be available depending on the language model provider.
  • Some scripts may fail or behave differently due to internal alignment prompts from Copilot.

  1. Install Visual Studio Code Insiders. It can be installed side-by-side with the stable version of Visual Studio Code.

  2. Open your project in Visual Studio Code Insiders (green icon).

  3. Download genaiscript.insiders.vsix from the releases page to your project folder.

    • …
    • genaiscript.insiders.vsix

  4. Right click on genaiscript.insiders.vsix and select Install from VSIX.

  5. Add a .vscode-insiders/argv.json file to enables loading the proposed APIs for the GenAIScript extension.

  6. Restart Visual Studio Code Insiders to enable the extension.

  7. When you try to run a script, you will be prompted to authorize the Copilot subscription. After authorizing access, you will see a quick pick dialog to select one of the installed Language Model.

    Note that this dialog is not used once you have a populated .env file.

If you are using the Insiders build, make sure to update the genaiscript.insiders.vsix regularly by repeating the steps above. The daily builds of Insiders may contain breaking chagnes that require new builds of our extension.
